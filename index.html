Fitting

2hr appointment to place

Let's return back to time ranges. For the start, we have a pretty basic need.
We have ranges available for booking and need to place 2hr appointment
A trick is to decrease length of each range by moving it's end edge back in time. If for some range end becomes to be earlier than start - pay attention to first block - we give up on it.
This way we are switching from "placing an event" to "placing a start point of the event". Then we can iterate over result and every point is valid place to start our 2hr event.

Merging

Let's go to the more complex stuff. Say, we have a set of events. Some are overlapping in time.

You know what? Let's think about them as "opening brace" and "closing brace". Then we could apply the same logic as we did with parentheses check. This time we don't need to verify validity, but instead when "sum" switches from 0 we remember this as a "start" point for merged interval. And ...

Fit sequence

  1. Type1 for 1hr
  2. Then type2 for 2hr
  3. Then type3 for 3hr
Most complex stuff ahead. Say, patient should have 3 different procedures of different lengthes. Side by side, back to back. And we need to figure out available slots.
Let's assume it's _availability ranges. In other words we _must_ place our appoinmtnet for each time somewhere inside.
First step we shrink every range by length of related appointment type.
  1. column 1 - for 1hr(note that 1 range will disappear completely)
  2. column 2 - for 2hr
  3. column 3 - for 3hr
As you remember, this is a shift from placing an event to placing its start point only.
Next step. We move all ranges backward for the sum of previous types.
What do I mean.
  1. Ranges in column 1 are moved for 0hr - no previous types
  2. Ranges in column 2 are moved for 1hr(Type1 length)
  3. Ranges in column 3 are moved for 3hr(Type1 length + Type2 length)
Now we will "calculate braces" but instead of creating new range when sum switches to/from 0 we will use 3 as threshold.
This way we will get ranges where _every_ type is avaialble.

Things we learned from parentheses check

  • How to switch from placing a range to placing just its start point
  • How to switch from placing several ranges back-to-back to placing just first of the sequence
  • How to invert and merge timeranges through iterating over +1 and -1

Important! I did not emphasize but it's critical to keep elements sorted in stable when converting ranges to edges.
